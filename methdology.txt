Sharpe ratio calculation: (Portfolio return - Risk-Free rate)/volatility

Equations for returns and growth for quantitative model

Daily return formula - runs performance regardless of price level
    rt = (Pt - Pt-1) / Pt-1
    rt  = return on day t
    Pt  = price of the asset on day t
    Pt-1 = price of the asset on the previous day before t

    CODE (data.py - to_period_returns):
        def to_period_returns(prices, frequency):
            period_prices = prices.resample(_resample_freq(frequency)).last()
            returns = period_prices.pct_change(fill_method=None).dropna(how="all")
            return returns
    HOW IT WORKS: pct_change() computes (Pt - Pt-1) / Pt-1 automatically for
    every row. If the S&P 500 was 4000 yesterday and 4040 today, the return is
    (4040 - 4000) / 4000 = 0.01 = +1%.

Cumulative return - this is the equation graphed against the S&P 500
    CR = Product of (1 + rt) for all t, minus 1
    CR: cumulative return
    rt: return at time t
    T: total number of trading days

    CODE (metrics.py):
        cumulative = (1 + returns).prod() - 1
    HOW IT WORKS: For each day's return, add 1 (so +1% becomes 1.01), then
    multiply them all together, then subtract 1. If your returns were +1%, +2%,
    -0.5%, the cumulative return = (1.01 * 1.02 * 0.995) - 1 = +2.5%.

Cumulative growth series - this is the actual line on the graph
    Growth(t) = Product of (1 + ri) for i = 1 to t
    This shows the growth of $1 invested at the start.

    CODE (quant.py - compute_strategy_series):
        def compute_strategy_series(returns, signal):
            if signal is None:
                strat_returns = returns
            else:
                strat_returns = returns * signal
            return (1 + strat_returns).cumprod()
    HOW IT WORKS: cumprod() builds the running product day by day. Day 1: 1.01.
    Day 2: 1.01 * 1.02 = 1.0302. Day 3: 1.0302 * 0.995 = 1.025. Each value
    tells you what $1 would be worth on that day. If signal is 0 (SELL), the
    return for that day is 0 (you are in cash and not affected by the market).

Annualized return - used to compare strategies over time
    Rannual = (1 + CR) ^ (252 / T) - 1
    Rannual = annualized return
    CR = cumulative return
    T = number of trading days
    252 = average number of trading days per year

    CODE (metrics.py):
        annualized = (1 + cumulative) ** (periods_per_year / len(returns)) - 1
    HOW IT WORKS: If you made 50% over 3 years, the annualized return is
    (1.50) ^ (1/3) - 1 = 14.5% per year. periods_per_year is 252 for daily,
    52 for weekly, 12 for monthly.

Volatility - measures how much returns fluctuate
    sigma = std(rt) * sqrt(252)
    sigma = volatility (annualized)
    rt = daily return
    std = standard deviation
    252 = trading days per year

    CODE (metrics.py):
        volatility = returns.std() * np.sqrt(periods_per_year)
    HOW IT WORKS: Standard deviation measures how spread out the returns are.
    A stock that goes +5%, -4%, +6%, -3% every day has high volatility. A stock
    that goes +0.1%, +0.2%, +0.1%, +0.1% has low volatility. Multiplying by
    sqrt(252) converts from daily volatility to yearly volatility.

Sharpe Ratio - better return per unit of risk
    S = (Rannual - Rf) / sigma_annual
    S = Sharpe ratio
    Rannual = annualized return
    Rf = risk-free rate (set to 2% = the return of US Treasury bonds)
    sigma_annual = annualized volatility

    CODE (metrics.py):
        rf_period = (1 + risk_free_annual) ** (1 / periods_per_year) - 1
        excess = returns - rf_period
        sharpe = excess.mean() / returns.std() * np.sqrt(periods_per_year)
    HOW IT WORKS: The Sharpe ratio answers "how much extra return did you get for
    each unit of risk?" A Sharpe of 1.0 means you got 1% extra return for every
    1% of volatility. Higher is better. The risk-free rate (2%) is subtracted
    because even a savings account earns that much, so only returns ABOVE that
    count as real performance.

Maximum drawdown - Measures worst loss from peak to bottom
    Max Drawdown = min over all t of (Value(t) / Peak(t) - 1)
    Peak(t) = highest portfolio value up to time t
    Value(t) = portfolio value at time t

    CODE (metrics.py):
        def max_drawdown(series):
            peak = series.cummax()
            drawdown = (series / peak) - 1
            return float(drawdown.min())
    HOW IT WORKS: cummax() tracks the highest value the portfolio has ever reached.
    If the portfolio was worth $1.50 at its peak and dropped to $1.20, the drawdown
    at that point is (1.20 / 1.50) - 1 = -20%. The max drawdown is the WORST such
    drop across the entire timeline. This shows how bad the worst crash was.

Portfolio return - preventing look-ahead bias
    rt_portfolio = Signal(t-1) * rt
    Signal(t-1) = trading decision from the PREVIOUS day (1 = invested, 0 = cash)
    rt = asset return on day t

    CODE (quant.py - quant_model_signals):
        def quant_model_signals(returns, model):
            signals = QUANT_MODELS[model](returns)
            return signals.shift(1).fillna(0.0)
    CODE (main.py - LLM signal alignment):
        aligned_signal = raw_signal.shift(1).fillna(0.0)
        llm_series[model] = compute_strategy_series(spy_returns, aligned_signal)
    HOW IT WORKS: shift(1) moves every signal forward by one day. This means the
    decision made on Monday is applied to Tuesday's return. The model NEVER sees
    tomorrow's data before making today's decision. This prevents "look-ahead bias",
    which is when a model accidentally cheats by using future information.

--- FILE 1: data.py (Data Collection) ---
PURPOSE: Loads S&P 500 price data and converts it into returns.

This is the first step in the entire project. Before any model can make
decisions, we need the raw price data for the S&P 500.

    # Load prices from a CSV file and filter to the date range
    def load_prices_csv(path, date_column, tickers, start, end):
        df = pd.read_csv(path)
        df[date_column] = pd.to_datetime(df[date_column])
        df = df.set_index(date_column).sort_index()
        df = df[(df.index >= pd.to_datetime(start)) & (df.index <= pd.to_datetime(end))]
        return df[tickers].dropna(how="all")

    Step by step:
    1. Read the CSV file (data/prices.csv) which has columns: date, SPX, AOR
    2. Convert the date column from text to actual dates
    3. Filter to only the dates between --start and --end
    4. Return only the columns we need (SPX for S&P 500, AOR for advisor proxy)

    # Convert raw prices into percent returns
    def to_period_returns(prices, frequency):
        period_prices = prices.resample(_resample_freq(frequency)).last()
        returns = period_prices.pct_change(fill_method=None).dropna(how="all")
        return returns

    Step by step:
    1. Resample to the chosen frequency (D=daily, W=weekly, M=monthly)
       - For monthly: takes the last price of each month
    2. pct_change() computes: (today's price - yesterday's price) / yesterday's price
    3. Drop any rows that are all NaN (missing data)

    The output is a table like:
    Date        | SPX     | AOR
    2020-01-31  | +0.004  | +0.002
    2020-02-29  | -0.083  | -0.052
    2020-03-31  | -0.125  | -0.089
    ...
    Each number is the percent return for that month.


--- FILE 2: quant.py (Quantitative Math Model) ---
PURPOSE: A pure math model that uses 4 financial indicators to decide
BUY or SELL each period. No AI, no opinions, just math rules.

THE 4 INDICATORS:

Indicator 1 - Trend (Moving Average Crossover):
    Formula: Trend = SMA(6) / SMA(12) - 1
    SMA(6) = average price over the last 6 periods
    SMA(12) = average price over the last 12 periods
    Rule: If the short-term average is ABOVE the long-term average, the
    market is trending upward. This is a BUY signal.

    Code:
        trend = prices.rolling(6).mean() / prices.rolling(12).mean() - 1
        trend_up = trend > 0

    Example: If the 6-month average price is $4200 and the 12-month average
    is $4100, then trend = (4200/4100) - 1 = +2.4%. This is positive, so
    trend_up = True.

Indicator 2 - Momentum (is the market going up recently?):
    Formula: Momentum = Pt / Pt-3 - 1
    Rule: If the price today is higher than 3 periods ago, the market has
    positive momentum. This is a BUY signal.

    Code:
        momentum = prices / prices.shift(3) - 1
        momentum_up = momentum > 0

    Example: If the price today is $4200 and 3 months ago it was $4000,
    then momentum = (4200/4000) - 1 = +5%. Positive, so momentum_up = True.

Indicator 3 - RSI (Relative Strength Index):
    Formula: RS = Average Gain / Average Loss (over 14 periods)
             RSI = 100 - (100 / (1 + RS))
    Rule: RSI ranges from 0 to 100. Above 70 means "overbought" (the market
    has gone up too fast and might crash). Below 70 is safe to BUY.

    Code:
        def _rsi(prices, window=14):
            delta = prices.diff()
            gain = delta.clip(lower=0).rolling(window).mean()
            loss = (-delta.clip(upper=0)).rolling(window).mean()
            rs = gain / loss.replace(0, pd.NA)
            return 100 - (100 / (1 + rs))

        rsi = _rsi(prices, window=14)
        rsi_ok = rsi < 70

    Example: If the average gain over 14 days is $20 and the average loss is $10,
    then RS = 20/10 = 2, and RSI = 100 - (100/3) = 66.7. Since 66.7 < 70,
    rsi_ok = True (safe to buy).

Indicator 4 - Volatility Filter (is the market too unstable?):
    Formula: Vol = std(returns) over 6 periods
    Rule: If current volatility is below the historical median (middle value)
    of volatility, the market is calm enough to invest.

    Code:
        volatility = returns.rolling(6).std()
        vol_ok = volatility < volatility.rolling(12).median()

    Example: If the current 6-month volatility is 12% and the 12-month median
    volatility is 15%, then vol_ok = True (market is calmer than usual).

THE DECISION RULE:
    Code:
        conditions_met = (trend_up.astype(int) + momentum_up.astype(int)
                          + vol_ok.astype(int) + rsi_ok.astype(int))
        signal = conditions_met >= 2

    If at least 2 out of 4 conditions are True, the quant model says BUY (1.0).
    Otherwise it says SELL (0.0) and stays in cash.

    Example decision:
        trend_up = True (1), momentum_up = True (1), vol_ok = False (0), rsi_ok = True (1)
        conditions_met = 1 + 1 + 0 + 1 = 3. Since 3 >= 2, signal = BUY.

PREVENTING LOOK-AHEAD BIAS:
    Code:
        def quant_model_signals(returns, model):
            signals = QUANT_MODELS[model](returns)
            return signals.shift(1).fillna(0.0)

    shift(1) moves every signal forward by one period. The decision computed on
    day t is applied to day t+1. The model never uses future data.


--- FILE 3: llm.py (Large Language Model Signals) ---
PURPOSE: Simulates what 5 different LLMs (GPT, Claude Sonnet, Gemini,
DeepSeek, Grok) would decide if given real S&P 500 market data.

HOW EACH LLM MAKES A DECISION:
    Each LLM is given the same market data that a real person or AI would see:
    - The last 5 periods of returns (did the market go up or down recently?)
    - The 20-day price trend (is the market in an uptrend or downtrend?)
    - Recent volatility (is the market stable or shaky?)

    Each LLM has a different "personality" defined by two parameters:
    - skill: how good it is at spotting danger (0 to 1, higher = better)
    - aggression: how eager it is to stay invested (0 to 1, higher = more invested)

    Code:
        model_profiles = {
            "gpt":            {"skill": 0.55, "aggression": 0.85},
            "claude_sonnet":  {"skill": 0.60, "aggression": 0.80},
            "gemini":         {"skill": 0.45, "aggression": 0.90},
            "deepseek":       {"skill": 0.65, "aggression": 0.75},
            "grok":           {"skill": 0.50, "aggression": 0.82},
        }

    What each personality means:
    - GPT: Aggressive investor (0.85), decent at spotting crashes (0.55)
    - Claude Sonnet: More cautious (0.80), better at spotting crashes (0.60)
    - Gemini: Very aggressive (0.90), weakest at spotting crashes (0.45)
    - DeepSeek: Most cautious (0.75), best at spotting crashes (0.65)
    - Grok: Middle of the road on both (0.82 aggression, 0.50 skill)

THE DECISION PROCESS (for each day):
    Code:
        buy_score = aggression   # Start biased toward investing

        # Step 1: Look at the last 5 days of returns
        recent_5 = returns.iloc[max(0, idx - 5):idx]
        recent_mean = recent_5.mean()
        recent_vol = recent_5.std()

        # Step 2: If recent returns are negative, skilled models pull back
        if recent_mean < -0.005:
            buy_score -= skill * 0.4
        if recent_mean < -0.015:
            buy_score -= skill * 0.3

        # Step 3: Check the 20-day trend
        trend = prices.iloc[idx] / prices.iloc[idx - lookback] - 1
        if trend < -0.02:
            buy_score -= skill * 0.3
        elif trend > 0.03:
            buy_score += 0.1

        # Step 4: If volatility is spiking, cautious models pull back
        if recent_vol > 0.02:
            buy_score -= skill * 0.2

        # Step 5: Add randomness (simulates LLM temperature)
        noise = rng.normal(0, 0.15)
        buy_score += noise

        # Step 6: Make the final decision
        buy_score = max(0.0, min(1.0, buy_score))
        signal = 1.0 if rng.random() < buy_score else 0.0

    Step by step example (DeepSeek, skill=0.65, aggression=0.75):
    - Start: buy_score = 0.75
    - Last 5 days averaged -2% return: buy_score -= 0.65 * 0.4 = -0.26 -> 0.49
    - That -2% is also < -1.5%: buy_score -= 0.65 * 0.3 = -0.195 -> 0.295
    - 20-day trend is -5%: buy_score -= 0.65 * 0.3 = -0.195 -> 0.10
    - Random noise of +0.05: buy_score -> 0.15
    - 15% chance of BUY -> probably SELL (stay in cash, dodge the crash)

    This is how the LLMs can beat the S&P 500: they sometimes dodge crashes.
    But they can also lose by sitting in cash during rallies.

WHY RESULTS ARE DIFFERENT EVERY RUN:
    Code:
        seed_base = int(time.time() * 1000) % (2**31)
        model_hash = sum(ord(c) for c in model_name)
        rng = np.random.default_rng(seed_base + model_hash)

    The random seed is based on the current time in milliseconds. Since the
    time is different every time you run the program, the random numbers are
    different, so the LLM decisions are different, and the final graph lines
    move to different positions each run.

THE PROMPT TEMPLATE (for live API mode):
    When using real LLM APIs, each model receives this exact prompt:

    Code:
        def _build_llm_prompt(current_date, recent_returns, trend_20d, volatility_20d):
            return (
                "You are a financial decision assistant.\n\n"
                f"Date: {current_date.date().isoformat()}\n"
                f"Last {len(returns_list)} days returns: {returns_list}\n"
                f"20-day trend (price vs 20D MA): {trend_20d:.2%}\n"
                f"20-day volatility: {volatility_20d:.2%}\n\n"
                "Based only on the information available up to today, should an "
                "investor be invested in the S&P 500 tomorrow?\n"
                "Respond with exactly one word: BUY, HOLD, or SELL."
            )

    The prompt only gives data up to today's date. It never gives the LLM
    tomorrow's prices. This prevents look-ahead bias.


--- FILE 4: main.py (The Backtest Engine - Compares Everything) ---
PURPOSE: This is the main file that runs the entire competition. It loads
data, runs all models, computes metrics, and generates the chart.

STEP 1 - Load price data and compute returns:
    Code:
        prices = load_prices_csv(config.prices_csv, ...)
        returns = to_period_returns(prices, config.frequency)
        spy_returns = returns[config.benchmark_ticker].dropna()

    This loads the S&P 500 prices from the CSV and converts them to returns.
    spy_returns is a list of how much the S&P went up or down each period.

STEP 2 - Create the advisor curve:
    Code:
        if config.advisor_mode == "synthetic":
            advisor_drag = config.advisor_underperformance_annual / periods_per_year
            advisor_returns = spy_returns - advisor_drag

        advisor_returns = apply_annual_fee(
            advisor_returns, config.advisor_fee_annual, config.frequency
        )

    Based on SPIVA scorecard data, financial advisors underperform the S&P 500
    by approximately 1.5% per year. On top of that, they charge a 1% annual
    management fee. So the advisor's return each period is:

        advisor_return = S&P_return - (1.5% / 12 months) - (1% / 12 months)

    This means each month, the advisor loses about 0.21% compared to just
    buying the S&P 500 directly.

STEP 3 - Generate LLM signals:
    Code:
        llm_signals = load_llm_signals(
            config.llm_signals_dir, config.frequency,
            returns_index=spy_returns.index, returns_data=spy_returns,
        )

    This calls the LLM simulation from llm.py. Each of the 5 LLMs looks at
    the S&P 500 data and produces a BUY (1) or SELL (0) decision for each
    period.

STEP 4 - Apply signals to get strategy returns (with look-ahead bias prevention):
    Code:
        for model, signal in llm_signals.items():
            raw_signal = signal.reindex(spy_returns.index).ffill().fillna(0.0)
            aligned_signal = raw_signal.shift(1).fillna(0.0)
            llm_series[model] = compute_strategy_series(spy_returns, aligned_signal)

    For each LLM:
    1. Take its BUY/SELL decisions
    2. shift(1) moves them forward by one period (prevents look-ahead bias)
    3. Multiply by S&P returns: if signal=1 (BUY), you earn the market return.
       If signal=0 (SELL), your return is 0 (you are in cash).
    4. cumprod() builds the growth curve showing what $1 would become.

STEP 5 - Build the S&P 500 benchmark and quant model:
    Code:
        base_series = {
            "S&P 500": compute_strategy_series(spy_returns, None),
            quant_label: quant_series,
        }
        if advisor_returns is not None:
            base_series["Advisor"] = compute_strategy_series(advisor_returns, None)
        series_df = pd.DataFrame({**base_series, **llm_series})

    The S&P 500 benchmark uses signal=None, meaning it is always invested
    (buy and hold). All strategies are combined into one table (series_df)
    where each column is a strategy and each row is a date.

STEP 6 - Find the winner:
    Code:
        sp500_final = series_df["S&P 500"].iloc[-1]
        final_values = series_df.iloc[-1]
        excess_over_sp = (final_values - sp500_final).drop("S&P 500")
        best_strategy = excess_over_sp.idxmax()

    Take the final value of each strategy and subtract the S&P 500's final value.
    Whoever has the highest excess is the winner.

STEP 7 - Create the Hybrid model (if needed):
    Code:
        non_advisor_excess = excess_over_sp.drop("Advisor", errors="ignore")
        best_signal_strategy = non_advisor_excess.idxmax()

        if best_signal_strategy in llm_series:
            hybrid_signal = align_signals_to_returns(
                llm_signals[best_signal_strategy], advisor_returns.index
            )
            hybrid_label = f"Hybrid (Advisor + {best_signal_strategy})"

        hybrid_series = compute_strategy_series(advisor_returns, hybrid_signal)

    From the methodology: "If the LLM or quant model manages to beat the
    advisor in profits, then a new model will be created."

    The Hybrid takes the BEST model's BUY/SELL signals and applies them to
    the ADVISOR's returns. This answers: "If a financial advisor used AI
    to tell them when to buy and sell, would they do better?"

STEP 8 - Compute all performance metrics:
    Code:
        for col in series_df.columns:
            metric_rows[col] = performance_metrics(
                returns_series.dropna(),
                config.risk_free_annual,
                config.frequency,
            )
        metrics_df = pd.DataFrame(metric_rows).T

    For every strategy (S&P 500, Advisor, GPT, Claude, Gemini, DeepSeek, Grok,

--- FILE 5: metrics.py (Performance Calculations) ---
PURPOSE: Computes all 5 performance metrics for each strategy.

    Full code with annotations:

        def performance_metrics(returns, risk_free_annual, frequency):
            # How many periods in a year (252 trading days, 52 weeks, 12 months)
            periods_per_year = {"D": 252, "W": 52, "M": 12}.get(frequency, 52)

            # Convert annual risk-free rate to per-period rate
            rf_period = (1 + risk_free_annual) ** (1 / periods_per_year) - 1

            # Excess returns = actual returns minus risk-free rate
            excess = returns - rf_period

            # METRIC 1: Cumulative return = total growth
            cumulative = (1 + returns).prod() - 1

            # METRIC 2: Annualized return = average yearly growth
            annualized = (1 + cumulative) ** (periods_per_year / len(returns)) - 1

            # METRIC 3: Volatility = how much returns bounce around (annualized)
            volatility = returns.std() * np.sqrt(periods_per_year)

            # METRIC 4: Sharpe ratio = return per unit of risk
            sharpe = excess.mean() / returns.std() * np.sqrt(periods_per_year)

            # METRIC 5: Max drawdown = worst peak-to-trough loss
            max_dd = max_drawdown((1 + returns).cumprod())

        def plot_excess_over_sp500(series, output_dir, show=False):
            # Subtract S&P 500 from every strategy
            excess = series.subtract(series["S&P 500"], axis=0)
            excess = excess.drop(columns=["S&P 500"])
